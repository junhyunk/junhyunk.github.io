### Part 4 를 읽고 성능테스트의 목적에 대해 정리하시오. (Word 2 page 분량)

1. 성능 테스트 목표 결정
    - 개요
        - 성능 테스트 목표를 결정하는 핵심적인 사항은 향후 개선을 위한 변화, 잠재적인 위협을 파악하는 것임.
        - 성능 테스트 목표는 프로젝트 초기에 식별하는 것이 좋지만, 주기적으로 팀 구성원에게 새로운 목표를 추가하고 싶은지 확인하는 것도 중요함.
        - 성능 테스트 목표를 결정하는 데 있어 고수준에서 고려해야 할 사항은 다음과 같음.
            1. 성능 테스트 목표는 성능 검증과 검증 활동의 시작점을 나타냄.
            2. 성능 테스트 목표는 비즈니스 진입점과 같이 시작함(비즈니스 규모, 향후 성장 등등).
            3. 성능 테스트 목표는 비즈니스 요구사항과 연관이 있으므로, 실세계의 사용자와 관련된 비즈니스 시나리오를 대변해야 함.
            4. 고수준의 목표를 결정한 이후, 기술에 구체적이도록 목표를 다듬을 수 있음.
    
    - 용어
        - 성능 테스트 목표
        - 성능 목표
        - 성능 타겟
        - 성능 한계(임계)점
        - 성능 예산

    - 성능 테스트 목표를 결정하는 것의 접근 방법
        1. 성능 테스트 노력(?) 간 목표 설정
        2. 자원 사용 목표와 한계점을 capture하거나 예상
        3. metric 식별
        4. 결과 공유
        5. 목표, 타겟, 예산을 바꾸는 것을 인지하고 있음

2. 최종 사용자 응답시간 목표 수치화
    - 개요
        - 성능 요구사항과 목표의 차이를 식별
        - 사용자는 성능 테스트의 결과를 모르고, 궁금해 하지도 않음. 그러나, 어플리케이션이 느리다는 것은 인지할 수 있음.
    
    - 용어
        - 성능 요구사항
            - 계약 수준에서 정의한 의무이고, 수정이 불가능함.
        - 성능 목표
            - 배포(출시)되기 이전에 만족해야 할 기준이지만, 특정 상황에서는 변경될 수 있음.

    - 어플리케이션의 성능에 대한 사용자의 인식은, 응답 시간 목표를 정량화하는 것에 밀접한 관련이 있음.
    - 어플리케이션의 사용자는 대부분 데이터가 화면에 표시되기까지 걸리는 시간, 처리량, DB가 제공해야 할 초당 트랜잭션 처리 개수 같은 사항들을 제시할 수 없음.
    - 그러나, 사용자들은 어플리케이션의 성능을 과거 경험, 태스크의 중요성, 기대하고 있는 사항에 관련지어 인지할 수 있음.
